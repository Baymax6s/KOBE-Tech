# KOBE-Tech

神戸電子専門学校 ソフト4の1回目のチーム制作で作成する Web アプリケーションのリポジトリです。

スクラム開発で、プロダクトバックログアイテム（PBI）をもとに機能を実装していきます。

## プロダクト概要

学生や教員が記事・質問・回答・制作物を投稿し、知識共有やフィードバックを行えるサービスを目指します。

主な利用者は以下を想定しています。

- 学生
- 教員
- 管理者
- 未ログインの閲覧者

## プロダクトバックログアイテム（PBI）

| No. | ユーザーストーリー |
| --- | --- |
| 1 | 閲覧者として投稿された記事の詳細な内容を参照したい。自分が知りたい情報があるか確認したいからだ。 |
| 2 | 回答者として投稿された質問に回答したい。自分の知識を他の学生に役立てたいからだ。 |
| 3 | 質問者として分からないことを質問でき、多くの人が見てくれる場所がほしい。たくさんの人から答えが欲しいからだ。 |
| 4 | 閲覧者として投稿された最新の記事を一覧で見たい。新しくどんな記事が投稿されているかを知りたいからだ。 |
| 5 | 閲覧者として記事に対してコメントを書きたい。記事の内容について質問や指摘をしたいからだ。 |
| 6 | 投稿者として自分の書いた記事への評価を知りたい。自分の記事の正確さや他者にわかりやすく伝わっているかを知って参考にしたいからだ。 |
| 7 | 閲覧者として分類別に記事や質問を閲覧したい。興味のある記事を簡単に探したいからだ。 |
| 8 | 閲覧者・質問者として優れた回答に対して高い評価をしたい。利用者全体に回答の価値を知らせたいからだ。 |
| 9 | 閲覧者として投稿者や回答者のプロフィールを知りたい。質問や回答の背景を知りたいからだ。 |
| 10 | 管理者としてユーザー（学生、教員）にパスワードを変更させたい。セキュリティを高めたいからだ。 |
| 11 | 投稿者として自分の制作物を公開できるようにしたい。自分の制作物に対して閲覧者の評価やアドバイスがほしいからだ。 |
| 12 | ユーザー（学生、教員）としてログイン・ログアウトしたい。Web サイトの機能を利用したいからだ。 |

## 技術構成

| 領域 | 技術 |
| --- | --- |
| フロントエンド | Vue 3 / Vite / TypeScript / Vuetify / Pinia |
| バックエンド | Go / Gin |
| データベース | PostgreSQL |
| API ドキュメント | Swagger / OpenAPI |
| 開発環境 | Docker Compose / mise |

## ディレクトリ構成

```text
.
├── api/  # Go + Gin のバックエンド
└── app/  # Vue 3 + Vite のフロントエンド
```

## セットアップ

バックエンドとフロントエンドで手順が分かれています。

- バックエンド: [api/README.md](./api/README.md)
- フロントエンド: [app/README.md](./app/README.md)

## 開発方針

- スクラム開発で進める
- PBI をもとにスプリントごとの実装対象を決める
- 実装前に目的と受け入れ条件を確認する
- API 変更時は OpenAPI 定義と TypeScript クライアントを更新する
- 小さく動く単位で実装し、レビューしやすい変更にする

## 現在の主な機能

- ログイン
- ログアウト
- 記事一覧表示
- 記事投稿
- 認証中ユーザー情報取得

## API 型生成

バックエンドの OpenAPI 定義を更新した場合は、フロントエンド側の API 型を再生成します。

```bash
cd app
npm run generate:api
```
